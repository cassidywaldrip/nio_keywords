# Museums & Cultural Institutions Keywords

Scripts and queries for collecting a global list of museum and cultural institution names from **Wikidata**, covering 17 institution types across all countries. Names are retrieved in English and local languages for use in a keyword collector pipeline.

## Data Source

All data is sourced from **Wikidata** ([wikidata.org](https://www.wikidata.org/)), a free, open knowledge base maintained by the Wikimedia Foundation. As of late 2025, Wikidata contains over 101,000 museum items alone.

Names are retrieved in multiple languages (English, Chinese, French, German, Spanish, Italian, Japanese, Korean, Arabic, Russian, Portuguese, and others depending on region), satisfying the requirement for both English and local-language versions.

## Files

| File | Description |
|------|-------------|
| `fetch_cultural_institutions.py` | **Main script.** Queries Wikidata's SPARQL endpoint for all 31 institution types with automatic pagination (5,000 items per page), retry logic, and caching. Outputs CSV files and a keyword list. |
| `cultural_institutions_optimized.md` | Ready-to-use SPARQL queries organized by institution type and continent. Optimized to avoid Wikidata Query Service timeout (no `wdt:P279*` subclass traversal; continent-level splits for large categories). Can be run manually at [query.wikidata.org](https://query.wikidata.org/). |
| `cultural_institutions_wikidata_queries.md` | Original (unoptimized) SPARQL queries. May timeout on large categories — use the optimized version or the Python script instead. |
| `cultural_institutions/` | (Generated by script) Individual CSV files per institution type, serving as cache so interrupted runs can resume. |
| `cultural_institutions_all.csv` | (Generated by script) Merged CSV with columns: `wikidata_id`, `name_en`, `alt_names`, `country`, `type`. |
| `cultural_institutions_keywords.txt` | (Generated by script) Final keyword list — one name per line, deduplicated, including both English and local-language names. |

## How to Use

### Option A: Python Script (Recommended)

Handles pagination, retries, rate limiting, and caching automatically.

```bash
pip install requests pandas
python fetch_cultural_institutions.py
```

The script queries each institution type separately, saves intermediate results to `cultural_institutions/`, and merges everything into the final outputs. If interrupted, re-running the script will skip already-cached types.

**Output columns in `cultural_institutions_all.csv`:**

| Column | Description |
|--------|-------------|
| `wikidata_id` | Wikidata item ID (e.g., Q160236) |
| `name_en` | English name |
| `alt_names` | Alternative names and local-language names (comma-separated) |
| `country` | Country where the institution is located |
| `type` | Institution type (e.g., `museum`, `opera_house`) |

### Option B: Manual SPARQL Queries

1. Go to [query.wikidata.org](https://query.wikidata.org/)
2. Copy a query from `cultural_institutions_optimized.md`
3. Click the blue play button to execute
4. Click "Download" → CSV

**If you get a timeout:** try the continent-level or country-level queries provided in the optimized file. The Python script (Option A) avoids this problem entirely.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `upstream request timeout` | Use the Python script instead of manual SPARQL, or split queries by country (country IDs listed in the optimized queries file) |
| `Query timeout limit reached` | Same as above — the query is too complex for Wikidata's 60-second limit |
| `HTTP 429 (rate limit)` | The Python script handles this automatically; if running manual queries, wait 30+ seconds between requests |
| Script interrupted mid-run | Just re-run it — cached results in `cultural_institutions/` will be reused |

## Notes

- Wikidata is a living database; results will vary slightly depending on when you run the queries.
- The `alt_names` field may contain names in dozens of languages, separated by commas. These are useful for multilingual keyword matching.
- Some institutions appear under multiple types (e.g., a venue tagged as both `museum` and `art_gallery`). The script deduplicates by `wikidata_id` in the final output.
